# Migration Guide: 5.x to 6.0

## Android Breaking Changes

### 1. Action Interceptor: Global to Per-Action

The global `setPaywallActionsInterceptor` is replaced by typed per-action interceptors. `processAction(Boolean)` is replaced by `PLYInterceptResult` enum.

**Before (v5):**

```kotlin
Purchasely.setPaywallActionsInterceptor { info, action, parameters, processAction ->
    when (action) {
        PLYPresentationAction.LOGIN -> {
            showLogin { success ->
                processAction(success)
            }
        }
        PLYPresentationAction.NAVIGATE -> {
            openUrl(parameters?.url)
            processAction(false)
        }
        else -> processAction(true)
    }
}
```

**After (v6):**

```kotlin
Purchasely.interceptAction<PLYPresentationAction.Login> { info ->
    val result = showLogin()
    if (result.success) {
        Purchasely.userLogin(result.userId) { true }
        PLYInterceptResult.SUCCESS
    } else {
        PLYInterceptResult.NOT_HANDLED
    }
}

Purchasely.interceptAction<PLYPresentationAction.Navigate> { info ->
    openUrl(info.action.url)
    PLYInterceptResult.SUCCESS
}
```

**Detection regex:**
```regex
setPaywallActionsInterceptor\s*\{
processAction\s*\(
```

---

### 2. PLYPresentationAction: Enum to Sealed Class

Action types change from enum constants to sealed class variants with parameters on each type.

**Before (v5):**

```kotlin
when (action) {
    PLYPresentationAction.PURCHASE -> { /* ... */ }
    PLYPresentationAction.RESTORE -> { /* ... */ }
    PLYPresentationAction.LOGIN -> { /* ... */ }
    PLYPresentationAction.CLOSE -> { /* ... */ }
    PLYPresentationAction.NAVIGATE -> { /* ... */ }
    PLYPresentationAction.OPEN_PRESENTATION -> { /* ... */ }
    PLYPresentationAction.PROMO_CODE -> { /* ... */ }
}
```

**After (v6):**

```kotlin
when (action) {
    is PLYPresentationAction.Purchase -> { /* action.plan, action.offer */ }
    is PLYPresentationAction.Restore -> { /* ... */ }
    is PLYPresentationAction.Login -> { /* ... */ }
    is PLYPresentationAction.Close -> { /* ... */ }
    is PLYPresentationAction.Navigate -> { /* action.url */ }
    is PLYPresentationAction.OpenPresentation -> { /* action.presentationId */ }
    is PLYPresentationAction.PromoCode -> { /* ... */ }
}
```

**Detection regex:**
```regex
PLYPresentationAction\.(PURCHASE|RESTORE|LOGIN|CLOSE|NAVIGATE|OPEN_PRESENTATION|PROMO_CODE)
```

---

### 3. PLYRunningMode.PaywallObserver Renamed to Observer

**Before (v5):**

```kotlin
Purchasely.Builder(applicationContext)
    .runningMode(PLYRunningMode.PaywallObserver)
    .build()
```

**After (v6):**

```kotlin
Purchasely.Builder(applicationContext)
    .runningMode(PLYRunningMode.Observer)
    .build()
```

**Detection regex:**
```regex
PLYRunningMode\.PaywallObserver
```

---

### 4. Deeplink API Renames

`isDeeplinkHandled` is renamed to `handleDeeplink`. `readyToOpenDeeplink` is renamed to `allowDeeplink`.

**Before (v5):**

```kotlin
val handled = Purchasely.isDeeplinkHandled(uri)
Purchasely.readyToOpenDeeplink = true
```

**After (v6):**

```kotlin
val handled = Purchasely.handleDeeplink(uri, activity)
Purchasely.allowDeeplink = true
```

**Detection regex:**
```regex
isDeeplinkHandled|readyToOpenDeeplink
```

---

### 5. User Attribute Methods Return Deferred<Boolean>

Attribute setters now return `Deferred<Boolean>` to indicate success/failure asynchronously.

**Before (v5):**

```kotlin
Purchasely.setUserAttribute("tier", "gold")  // fire-and-forget
```

**After (v6):**

```kotlin
val result: Deferred<Boolean> = Purchasely.setUserAttribute("tier", "gold")
// Optionally await the result
lifecycleScope.launch {
    val success = result.await()
    if (!success) {
        Log.w("PLY", "Failed to set attribute")
    }
}
```

**Detection regex:**
```regex
Purchasely\.setUserAttribute\(
```

---

### 6. Logging: Custom Loggers Receive ALL Messages

In v5, custom loggers only received messages at or above the configured log level. In v6, custom loggers receive ALL messages regardless of log level. A new `logcatEnabled` flag controls Logcat output separately.

**Before (v5):**

```kotlin
Purchasely.Builder(applicationContext)
    .logLevel(LogLevel.WARN)  // Custom logger only gets WARN+
    .build()
```

**After (v6):**

```kotlin
Purchasely.Builder(applicationContext)
    .logLevel(LogLevel.WARN)
    .logcatEnabled(false)  // Disable Logcat; custom logger gets ALL messages
    .build()
```

**Detection regex:**
```regex
\.logLevel\(LogLevel\.
```

---

### 7. Gradle and Kotlin Version Requirements

v6 requires Gradle 9.3.0+ and Kotlin 2.2.x+.

**Before (v5):**

```kotlin
// gradle-wrapper.properties
distributionUrl=https\://...gradle-8.x-bin.zip

// build.gradle.kts
kotlin("android") version "1.9.x"
```

**After (v6):**

```kotlin
// gradle-wrapper.properties
distributionUrl=https\://...gradle-9.3.0-bin.zip

// build.gradle.kts
kotlin("android") version "2.2.0"
```

**Detection regex:**
```regex
gradle-[0-8]\.\d+|kotlin.*version.*["']1\.\d+
```

### 8. New PLYPresentation Builder Pattern

A new builder DSL for presentation display, with preload support.

**Before (v5):**

```kotlin
Purchasely.fetchPresentation(
    placementId = "ONBOARDING",
    callback = object : PLYPresentationCallback {
        override fun onPresentationFetched(presentation: PLYPresentation) {
            presentation.display(context)
        }
        override fun onPresentationClosed() {}
        override fun onPurchaseResult(result: PLYPurchaseResult) {}
    }
)
```

**After (v6):**

```kotlin
PLYPresentation {
    placementId("ONBOARDING")
    contentId("optional_content_id")
}.preload { presentation ->
    // Presentation preloaded and ready
}.display(context)
```

**Note:** The v5 `fetchPresentation` API still works in v6 but the builder pattern is the recommended approach.

### 9. PLYProductViewResult Deprecated

`PLYProductViewResult` is deprecated. Use `PLYPurchaseResult` from `PLYPresentationOutcome` instead.

**Before (v5):**

```kotlin
override fun onProductViewResult(result: PLYProductViewResult) {
    when (result) {
        PLYProductViewResult.PURCHASED -> { /* ... */ }
        PLYProductViewResult.CANCELLED -> { /* ... */ }
        PLYProductViewResult.RESTORED -> { /* ... */ }
    }
}
```

**After (v6):**

```kotlin
override fun onPresentationOutcome(outcome: PLYPresentationOutcome) {
    when (outcome.result) {
        PLYPurchaseResult.PURCHASED -> { /* ... */ }
        PLYPurchaseResult.CANCELLED -> { /* ... */ }
        PLYPurchaseResult.RESTORED -> { /* ... */ }
    }
}
```

**Detection regex:**
```regex
PLYProductViewResult|onProductViewResult
```

---

### 10. New allowCampaigns Flag

A new flag to control whether the SDK displays campaigns (win-back, retention, etc.).

**Before (v5):**

Campaigns were always displayed when configured. No programmatic control.

**After (v6):**

```kotlin
// Disable campaigns programmatically
Purchasely.allowCampaigns = false

// Re-enable campaigns
Purchasely.allowCampaigns = true
```

This is useful for suppressing campaigns during onboarding or specific user flows.

## iOS Deprecations (Non-Breaking)

iOS v6 introduces deprecations with backward-compatible shims. Existing code continues to work but should be updated.

### 1. isDeeplinkHandled to handleDeeplink

**Before (deprecated, shim exists):**

```swift
let handled = Purchasely.isDeeplinkHandled(url)
```

**After (recommended):**

```swift
let handled = Purchasely.handleDeeplink(url)
```

**Detection regex:**
```regex
isDeeplinkHandled
```

### 2. readyToOpenDeeplink to allowDeeplink

**Before (deprecated, shim exists):**

```swift
Purchasely.readyToOpenDeeplink = true
```

**After (recommended):**

```swift
Purchasely.allowDeeplink(true)
```

**Detection regex:**
```regex
readyToOpenDeeplink
```

### 3. Presentation Info Property Access Changes

Info object properties are accessed through the `presentation` sub-object.

**Before (deprecated, shim exists):**

```swift
let presentationId = info.presentationId
let placementId = info.placementId
let abTestId = info.abTestId
```

**After (recommended):**

```swift
let presentationId = info.presentation?.id
let placementId = info.presentation?.placementId
let abTestId = info.presentation?.abTestId
```

**Detection regex:**
```regex
info\.presentationId|info\.placementId|info\.abTestId
```

## Migration Checklist

### Android (Breaking -- must fix before upgrading)

- [ ] Update Gradle to 9.3.0+
- [ ] Update Kotlin to 2.2.x+
- [ ] Replace `setPaywallActionsInterceptor` with typed `interceptAction<Type>` calls
- [ ] Replace `PLYPresentationAction.PURCHASE` etc. with sealed class variants (`PLYPresentationAction.Purchase`)
- [ ] Replace `PLYRunningMode.PaywallObserver` with `PLYRunningMode.Observer`
- [ ] Replace `isDeeplinkHandled` with `handleDeeplink`
- [ ] Replace `readyToOpenDeeplink` with `allowDeeplink`
- [ ] Update user attribute calls to handle `Deferred<Boolean>` return type
- [ ] Replace `PLYProductViewResult` with `PLYPurchaseResult` / `PLYPresentationOutcome`
- [ ] Review custom logging setup for new `logcatEnabled` flag
- [ ] Consider adopting `PLYPresentation` builder pattern
- [ ] Consider using `allowCampaigns` flag if needed

### iOS (Non-breaking -- can update gradually)

- [ ] Replace `isDeeplinkHandled` with `handleDeeplink`
- [ ] Replace `readyToOpenDeeplink` with `allowDeeplink`
- [ ] Update `info.presentationId` to `info.presentation?.id` (and similar)
